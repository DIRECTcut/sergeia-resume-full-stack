# Run Hr Breaker

## Overview

Use this skill for the repo-local `hr-breaker` workflow. The repository already provides the supported entrypoints:

- `scripts/setup-hr-breaker.sh`
- `scripts/run-hr-breaker.sh`

Do not reimplement the setup or optimize flow inline when these wrappers fit the task.

## Workflow

### 1. Verify Inputs

Confirm the task includes:

- a source resume PDF
- a vacancy file, usually Markdown
- an optional language code, defaulting to `en`

If a path is unclear, inspect the repo and use the actual file path instead of guessing.

### 2. Prepare The Tool

Run:

```bash
scripts/setup-hr-breaker.sh
```

What it does:

- clones `btseytlin/hr-breaker` into `tools/hr-breaker` if missing
- checks out the pinned commit from the wrapper script
- runs `uv sync`
- creates `tools/hr-breaker/.env` from `.env.example` when needed

Do not print secret values. If credentials are needed, rely on the repo-local `.env` or `tools/hr-breaker/.env` without echoing them.

### 3. Run Optimization

Run:

```bash
scripts/run-hr-breaker.sh "<resume-pdf>" "<vacancy-file>" [language]
```

Example from this repo:

```bash
scripts/run-hr-breaker.sh "resumes/Sergei Afonchenko - Backend Developer.pdf" "tmp/ggr-global-senior-go-developer.md" en
```

Notes:

- `scripts/run-hr-breaker.sh` sources the repo-local `.env` automatically if it exists.
- Input should be an existing resume PDF, not a DOCX file.
- Output is generated by `hr-breaker` under `tools/hr-breaker/output/`.

### 4. Validation

Before removing or replacing any existing documentation, run the wrapper end-to-end with the user-specified sample inputs when policy permits.

Validation is complete only when:

- setup succeeds
- the optimize command exits successfully
- a new output PDF appears under `tools/hr-breaker/output/`

If a live run would send resume or vacancy contents to an external LLM/API, explain that risk briefly and get explicit user approval before proceeding.

## Done Criteria

- The local wrapper command is the documented source of truth.
- The command uses real repo paths.
- Secrets were not printed.
- Any cleanup of `README.md` happens only after validation.
